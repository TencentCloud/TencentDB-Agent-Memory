## 一句话：从哪开始看

**入口文件是 `index.ts`（根目录）。** 它是插件的"大门"，所有功能从这里连线。看完它你就知道整个项目的骨架了。

## 从哪开始看？推荐阅读顺序

### 如果你只想了解架构

```
1. index.ts                ← 看插件怎么注册，哪些钩子
2. src/core/tdai-core.ts   ← 看总调度器有哪些方法
3. src/core/hooks/auto-capture.ts  ← 看对话怎么录
4. src/core/hooks/auto-recall.ts   ← 看记忆怎么召
```

### 如果你想深入存储层

```
1. src/core/store/types.ts   ← 先看接口定义
2. src/core/store/sqlite.ts  ← 看默认实现（你刚修过的文件）
3. src/core/store/factory.ts ← 看怎么选后端
```

### 如果你想理解 L1/L2/L3 管道

```
1. src/core/record/l1-extractor.ts  ← L1 提取
2. src/core/scene/scene-extractor.ts   ← L2 场景
3. src/core/persona/persona-generator.ts ← L3 画像
4. src/utils/pipeline-manager.ts     ← 触发调度
```

## 三个关键对象

整个项目围绕三个核心对象运转：

### 1. `TdaiCore`（总调度器）
```typescript
// 只有一个实例，所有操作都通过它
class TdaiCore {
  handleBeforeRecall()   // 对话前：召回记忆
  handleTurnCommitted()  // 对话后：录制 + 触发管道
  searchMemories()       // 搜索 L1 记忆
  searchConversations()  // 搜索 L0 对话
}
```
源码：[src/core/tdai-core.ts](../src/core/tdai-core.ts)

### 2. `IMemoryStore`（存储接口）
```typescript
// 抽象存储层，SQLite 和 TCVDB 都实现这个接口
interface IMemoryStore {
  upsertL1()      // 写入 L1 记忆
  searchL1Vector() // 向量搜索 L1
  searchL1Fts()   // 关键词搜索 L1
  upsertL0()      // 写入 L0 对话
  // ...
}
```
源码：[src/core/store/types.ts](../src/core/store/types.ts)

### 3. `PipelineManager`（管道调度器）
```typescript
// 决定什么时候触发 L1 提取、L2 归纳、L3 画像
class MemoryPipelineManager {
  notifyTurnCommitted()  // 收到一轮对话，判断是否该跑管道
  scheduleL1()           // 触发 L1 提取
  scheduleL2()           // 触发 L2 场景归纳
  schedulePersona()      // 触发 L3 画像更新
}
```
源码：[src/utils/pipeline-manager.ts](../src/utils/pipeline-manager.ts)
