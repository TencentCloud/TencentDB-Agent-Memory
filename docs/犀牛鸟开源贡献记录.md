### PR #361 — fix(recall): optimize prompt cache hit rate

- **关联 Issue**：[#120](https://github.com/TencentCloud/TencentDB-Agent-Memory/issues/120)
- **分支**：`fix/issue-120-prompt-cache-optimization`
- **提交时间**：2026-07-02
- **状态**：🟢 Open
- **链接**：https://github.com/TencentCloud/TencentDB-Agent-Memory/pull/361

**问题根因**：
1. 主因：prependContext（动态 L1 记忆）每轮注入到用户消息前缀 → prefix-matching 缓存失效
2. 次因：appendSystemContext（稳定 persona/scene）放在 CACHE_BOUNDARY 之后 → 未参与缓存

**变更摘要**：

| 模块 | 说明 |
|------|------|
| `src/config.ts` | 新增 3 个 recall 配置项：injectionMode、showInjected、cacheDiagnostics |
| `openclaw.plugin.json` | configSchema 新增 3 个字段 |
| `index.ts` | before_prompt_build hook 路由：injectionMode=append 时动态召回走 appendContext；稳定内容映射到 prependSystemContext（CACHE_BOUNDARY 之前）；before_message_write 尊重 showInjected |
| `src/utils/recall-injection.ts` | strip/has 辅助函数（可测试） |
| `src/utils/cache-diagnostics.ts` | PrefixShape + CacheDiagnosticsTracker（缓存形状诊断） |
| 5 个测试文件 | 66 个新测试，包括确定性缓存模拟（hash-based prefix stability verification）⭐ |

**总计**：13 个文件，+1727 / -28 行

**核心差异化**：
- 缓存模拟测试：模拟 8 轮对话，用 SHA-256 计算 prompt 前缀哈希，验证 injectionMode=append 时系统前缀稳定、prepend 模式每轮变化
- Token 节省量估算：append 模式可节省 87.5% 的系统 prompt token 重复计算
- 融合了 PR #346、#350、#358、#343 的优点于一体

**提交**：
1. `2d48306` — fix(recall): optimize prompt cache hit rate for OpenAI-compatible providers (Fixes #120)
