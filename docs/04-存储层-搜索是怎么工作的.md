# 04 — 存储层：搜索是怎么工作的

> 阅读时间：~25 分钟 | 需要 SQL 基础概念

---

## 先搞清楚：数据存在哪

项目有两个存储后端可选，默认用 SQLite：

```
SQLite (默认)                          TCVDB (可选)
─────────────────                      ─────────────────
本地文件 vectors.db                    腾讯云远端向量数据库
零配置，开箱即用                        需要 URL + API Key
支持 FTS5 全文搜索                      支持原生混合搜索
支持 vec0 向量搜索（sqlite-vec 扩展）     服务端自带 embedding
适合单机 / 开发                          适合生产 / 多实例
```

本文专注讲 **SQLite 后端**，这也是你修过代码的地方。

---

## SQLite 数据库长什么样

文件路径：`~/.openclaw/state/memory-tdai/vectors.db`

里面有这些表：

```
vectors.db
├── l0_conversations    ← L0 对话元数据
├── l0_vec              ← L0 对话向量（vec0 虚拟表）
├── l0_fts              ← L0 对话全文索引（FTS5 虚拟表）
├── l1_records          ← L1 记忆元数据
├── l1_vec              ← L1 记忆向量（vec0 虚拟表）
├── l1_fts              ← L1 记忆全文索引（FTS5 虚拟表）
└── embedding_meta      ← 记录用的 embedding 模型信息
```

## 三种搜索方式

| 搜索方式 | 靠什么 | 适合什么 | 需要什么 |
|----------|--------|----------|----------|
| **FTS5** | SQLite 全文索引，BM25 评分 | 精确关键词匹配 | 无（SQLite 自带） |
| **Vector** | vec0 虚拟表，余弦相似度 | 语义相似搜索 | embedding 服务 |
| **Hybrid** | FTS5 + Vector 结果用 RRF 融合 | 综合最佳 | embedding 服务 |

## 搜索一：FTS5 关键词搜索

**代码位置**：[src/core/store/sqlite.ts](../src/core/store/sqlite.ts) — `buildFtsQuery()` 和 `searchL1Fts()`

### 一步步看

#### 第 1 步：把用户输入变成 FTS5 查询

```typescript
// sqlite.ts:241
export function buildFtsQuery(raw: string): string | null {
  // 第一步：清洗（你新加的）
  const cleaned = sanitizeFts5Input(raw);
  if (!cleaned) return null;

  // 第二步：分词
  const jieba = getJieba();
  if (jieba) {
    // 有 jieba → 中文精确分词
    // "用户喜欢TypeScript"
    //   → ["用户", "喜欢", "TypeScript"]
    tokens = jieba.cutForSearch(cleaned, true)
      .filter(t => /[\p{L}\p{N}]/u.test(t))  // 去标点
      .filter(t => !ZH_STOP_WORDS.has(t));    // 去停用词
  } else {
    // 没有 jieba → 用 Unicode 正则
    // "用户喜欢TypeScript"
    //   → ["用户喜欢TypeScript"]  ← 整个当做一个 token
    tokens = cleaned.match(/[\p{L}\p{N}_]+/gu) ?? [];
  }

  // 第三步：每个 token 加双引号，用 OR 连接
  // → '"用户" OR "喜欢" OR "TypeScript"'
  return tokens.map(t => `"${t.replaceAll('"', '')}"`).join(" OR ");
}
```

**为什么要把 token 加双引号？**

FTS5 里，双引号包裹的内容是**字面量短语查询**。也就是说：
- `用户`（不加引号） → FTS5 会 tokenize "用户" 然后搜 "用" 或 "户"
- `"用户"`（加引号） → FTS5 当做完整短语，必须精确匹配 "用户"

这是防御的第一层——即使用户输入了 `AND`/`OR`/`NOT`，它们在引号里就是普通文字。

**`sanitizeFts5Input` 干什么？**

这是你新加的第二层防御。在分词之前，把 FTS5 保留字和特殊字符从原始输入中直接移除：

```typescript
// sqlite.ts:210
export function sanitizeFts5Input(raw: string): string {
  return raw
    .replace(/\b(AND|OR|NOT|NEAR)\b/gi, " ")   // 去保留字
    .replace(/[*()"^{}]/g, " ")                  // 去特殊字符
    .replace(/\s+/g, " ")                        // 压缩空白
    .trim();
}
```

输入 `"hello AND world"` → 输出 `"hello world"`，然后把 `hello` 和 `world` 分别加引号，safe。

#### 第 2 步：把查询送给 SQLite

```typescript
// sqlite.ts — searchL1Fts()
searchL1Fts(ftsQuery: string, limit = 20): FtsSearchResult[] {
  // 用预编译的 prepared statement 执行
  const rows = this.stmtL1FtsSearch.all(ftsQuery, limit);
  // SQL: SELECT ... FROM l1_fts WHERE l1_fts MATCH ? ORDER BY rank LIMIT ?
  // BM25 自动评分，rank 越低越相关
  return rows.map(r => ({
    ...r,
    score: bm25RankToScore(r.rank)  // 把负的 BM25 rank 转成 0~1 分数
  }));
}
```

#### 第 3 步：FTS5 怎么建索引

写到 `l1_fts` 表时，不是直接存原文，而是先经过 **jieba 分词**：

```typescript
// sqlite.ts:253
export function tokenizeForFts(raw: string): string {
  const jieba = getJieba();
  if (!jieba) return raw;  // 没有 jieba 直接存原文

  // "用户五月去日本旅行" → "用户 五月 去 日本 旅行"
  const tokens = jieba.cutForSearch(raw, true);
  return tokens.join(" ");
}
```

分词后的文本用空格连接，FTS5 的 `unicode61` tokenizer 会按空格拆分，正好每个词对应一个 FTS5 token。

### 完整链路回顾

```
用户输入         buildFtsQuery     FTS5 MATCH        SQLite          结果
─────────        ─────────────     ──────────        ──────          ────
"我爱TypeScript"  → sanitize        → l1_fts         → BM25 排名      → 前 20 条
                  → jieba 分词        MATCH            → rank→score     记忆
                  → 加引号 OR 连接    '"我" OR "爱"
                                     OR "TypeScript"'
```

## 搜索二：Vector 语义搜索

**代码位置**：[src/core/store/sqlite.ts](../src/core/store/sqlite.ts) — `searchL1Vector()`

### 向量是什么

把一段文字变成一个数字数组（比如 [0.12, -0.34, 0.78, ...] × 1536 维）。两段意思相近的文字，它们的向量在空间中也相近。

```
"我喜欢 TypeScript"      → embedding → [0.1, 0.3, -0.2, ...]
"我爱用 TypeScript 编程"  → embedding → [0.15, 0.28, -0.18, ...]
                                   余弦相似度 = 0.95  ← 很接近！

"今天天气真好"           → embedding → [-0.4, 0.1, 0.6, ...]
                                   余弦相似度 = 0.12  ← 不相关
```

### 向量怎么来的

**代码位置**：[src/core/store/embedding.ts](../src/core/store/embedding.ts)

调用远端 API（OpenAI 兼容接口）把文字变成向量：

```typescript
// embedding.ts
async embed(texts: string[]): Promise<Float32Array[]> {
  const response = await fetch(`${baseUrl}/embeddings`, {
    body: JSON.stringify({
      model: "text-embedding-3-small",
      input: texts,
      dimensions: 1536,
    }),
  });
  return response.data.map(d => new Float32Array(d.embedding));
}
```

需要配置 `openclaw.json`：
```json
{
  "memory-tencentdb": {
    "embedding": {
      "provider": "openai",
      "baseUrl": "https://api.openai.com/v1",
      "apiKey": "${OPENAI_API_KEY}",
      "model": "text-embedding-3-small",
      "dimensions": 1536
    }
  }
}
```

如果不配，`provider = "none"`，则没有向量搜索，只能用 FTS5。

### 向量怎么搜

SQLite 用 sqlite-vec 扩展的 vec0 虚拟表实现 KNN（K 最近邻）搜索：

```typescript
// sqlite.ts — searchL1Vector()
searchL1Vector(queryEmbedding: Float32Array, topK = 5): VectorSearchResult[] {
  // 1. 多取 10 条来过滤零向量（旧数据的遗留）
  const rows = this.stmtSearchVec!.all(
    Buffer.from(queryEmbedding.buffer),  // 查询向量
    topK + 10                             // K 值
  );
  // SQL: SELECT record_id, distance FROM l1_vec
  //      WHERE embedding MATCH ? AND k = ? ORDER BY distance

  // 2. 过滤掉距离为 null/NaN 的结果（零向量导致）
  // 3. 读元数据表获取内容
  // 4. distance → score: score = 1.0 - distance
  // 5. 截断到 topK 返回
}
```

### distance 和 score 的关系

vec0 返回的是 **余弦距离**（cosine distance），范围 [0, 2]：
- 距离 0 = 两个向量方向完全相同 → score = 1.0（最相关）
- 距离 1 = 正交 → score = 0（不相关）
- 距离 2 = 方向完全相反 → score = -1（负相关）

代码中 `score = 1.0 - distance`。

## 搜索三：Hybrid 混合搜索

**代码位置**：
- [src/core/tools/memory-search.ts](../src/core/tools/memory-search.ts)
- [src/core/store/search-utils.ts](../src/core/store/search-utils.ts)

### 为什么要混合

FTS5 搜关键词很准，但同义词不通（搜"开心"找不到"高兴"）。
Vector 搜语义很强，但可能把不相关的也搜出来。

混在一起 => 取长补短。

### 混合方式：RRF（Reciprocal Rank Fusion）

```typescript
// search-utils.ts
export function rrfMerge(lists, getId, k = 60) {
  // 每条记录的 RRF 分数 = 所有列表中 1/(k + rank + 1) 的和
  // 出现在多个列表中 → 分数累积 → 排名更高
  const map = new Map();
  for (const list of lists) {
    for (let rank = 0; rank < list.length; rank++) {
      const score = 1 / (k + rank + 1);
      map.set(id, (map.get(id) || 0) + score);
    }
  }
  // 按 RRF 分数降序排列
  return sortedByRRFScore(map);
}
```

### 实际调用

```typescript
// memory-search.ts — 并行跑两种搜索
const [ftsItems, vecItems] = await Promise.all([
  // FTS5 关键词搜索
  (async () => {
    const ftsQuery = buildFtsQuery(query);
    if (!ftsQuery) return [];
    return await store.searchL1Fts(ftsQuery, candidateK);
  })(),
  // 向量语义搜索
  (async () => {
    const embedding = await embed(query);
    return await store.searchL1Vector(embedding, candidateK);
  })(),
]);

// RRF 融合
const merged = rrfMerge([ftsItems, vecItems], item => item.record_id);
return merged.slice(0, limit);
```

## FTS5 vs Vector vs Hybrid 选哪个

| 场景 | 推荐 | 原因 |
|------|------|------|
| 没配 embedding 服务 | FTS5 | 唯一可用选项 |
| 英文为主 | Hybrid | 英文关键词 + 语义都有效 |
| 中文为主 | Hybrid | jieba 分词 + 语义互补 |
| 只搜精确关键词 | FTS5 | 更快，不需要调 API |
| 只搜语义相似 | Vector | 更省 token |
| 不确定 | **Hybrid** | 默认推荐 |

配置方式：
```json
{
  "recall": {
    "strategy": "hybrid"  // 或 "keyword" 或 "embedding"
  }
}
```

## 你修的代码在整个搜索链路中的位置

```
用户输入
  │
  ▼
sanitizeFts5Input()   ← 你新加的！
  │
  ▼
buildFtsQuery()        ← 你修改的！"raw" → "cleaned"
  │
  ▼
searchL1Fts()          ← 送给 SQLite FTS5
  │
  ▼
BM25 排名 + 返回结果
```

你的修改确保了：**无论用户输入什么，FTS5 查询永远不会被意外操作符改变语义**。
